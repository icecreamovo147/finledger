# SPEC: FinLedger — 广告公司记账软件

## 1. Background & Goals

### Background
一家广告公司（业务涵盖文印、广告设计、物料制作、安装等）的老板需要一款内部提效的桌面端收入记账软件。当前主要痛点是：承接多家客户公司的业务时需要清楚记录每家公司的赊账情况，并在月底导出账单去催款结算。

### Goals
- 按客户公司分账本，独立管理各公司的收入流水
- 完整记录每一笔收入的明细（类别、金额、数量、尺寸、图片凭证等）
- 追踪每笔账目的结算状态（未结清 / 已结清），支持记录收款日期和收款方式
- 支持按账本导出 Excel 账单（全部未结清或勾选部分），用于找客户催款
- 首页看板展示关键经营数据
- 支持数据备份，保证数据安全

### Success Criteria
- 老板能在一分钟内完成一笔收入记录的录入
- 筛选未结清记录并导出 Excel 的操作链不超过三步
- 已结清记录不可被修改，保障账目安全
- 单机运行稳定，数据不丢

## 4. User Stories

- **US1**: 作为管理员，我要通过用户名和密码登录系统，以便保护账目数据不被未授权访问。
- **US2**: 作为管理员，我要在首次启动时创建管理员账号，以便建立系统的初始访问凭证。
- **US3**: 作为管理员，我要创建、删除用户账号和管理密码，以便管理谁可以访问系统。
- **US4**: 作为老板，我要创建一个新的客户公司账本，以便独立管理该公司的所有收入记录。
- **US5**: 作为老板，我要在某个账本中快速录入一笔收入（选类别、填金额、传图片），以便记录每一次交易。
- **US6**: 作为老板，我要看到某个账本下哪些账还没结清，以便知道需要找哪些客户催款。
- **US7**: 作为老板，我要把某个账本中的未结清记录导出成 Excel 文件，以便发给客户核对和催款。
- **US8**: 作为老板，当客户付款后，我要把对应记录标记为"已结清"并记录收款日期和方式，以便追踪回款情况。
- **US9**: 作为老板，我希望已结清的记录被锁定不能修改，以防误操作导致账目不一致。
- **US10**: 作为老板，我要在首页看到本月的收入概览和哪些客户欠款最多，以便掌握经营状况。
- **US11**: 作为老板，我要定期备份数据，以便电脑出问题时能恢复。

## 5. Boundary Conditions & Constraints

### Data Constraints
- 账本数量无上限
- 每个账本下的记录数量无上限
- 每笔记录可上传任意数量图片
- 图片存储在本地文件系统，数据库仅存路径引用
- 金额精度：两位小数（元）
- 数量精度：整数

### Business Rules
- **认证规则**：所有功能必须在登录后才可访问。密码使用 bcrypt/argon2 哈希存储，不存储明文。
- **登录态持久化**：支持"记住我"功能，关闭应用后重新打开在有效期内无需重新登录。
- **首次初始化**：应用首次启动检测是否存在用户账号，无用户时自动进入初始化向导。
- **结算锁定规则**：记录一旦标记为"已结清"即进入锁定状态，不可编辑、不可删除
- **不支持部分付款**：一笔记录要么全未结清，要么全已结清，无中间状态
- **收款方式**：记录收款日期（必填）+ 收款方式（必填，预设：现金/银行转账/微信/支付宝，支持自定义输入）
- **结算回退**：已结清的记录如需修改，必须先回退为"未结清"状态（操作需二次确认）

### Platform Constraints
- 桌面端应用（macOS / Windows）
- 单机运行，SQLite 本地数据库
- 数据库结构预留未来局域网协作扩展

### Data Backup
- 手动触发备份，将 SQLite 数据库文件复制到用户指定目录
- 支持从备份文件恢复（覆盖当前数据库）

## 6. Error & Exception Scenarios

| 场景 | 处理方式 |
|------|----------|
| 未登录直接访问功能页面 | 重定向到登录页 |
| 用户名或密码错误 | 提示"用户名或密码错误"，不清空用户名输入框 |
| 多次登录失败 | 连续 5 次失败后锁定 15 分钟，提示剩余等待时间 |
| 首次启动无用户账号 | 自动跳转初始化向导，强制创建管理员账号 |
| 图片文件被手动删除 | 记录详情中图片显示为"文件不存在"占位符，不影响其他数据 |
| 导出的 Excel 目标路径无写入权限 | 提示用户选择其他路径 |
| 数据库文件被占用或损坏 | 启动时检测数据库完整性，异常时提示并引导恢复备份 |
| 备份时磁盘空间不足 | 提示磁盘空间不足，备份失败 |
| 尝试编辑已结清记录 | 按钮置灰不可点击，悬停显示"已结清记录不可修改" |
| 删除账本（含有记录） | 二次确认弹窗，提示将删除该账本下所有记录 |
| 删除用户账号 | 二次确认弹窗，不允许删除当前登录用户自己 |
| 图片文件过大 | 单张图片限制 20MB，超过时前端拦截并提示 |

## 7. Non-Functional Requirements

### Performance
- 应用冷启动时间 ≤ 3 秒
- 记录列表加载（1000 条以内）≤ 500ms
- 图片缩略图加载使用懒加载

### Security
- 密码使用 bcrypt 哈希存储，不可逆
- SQLite 数据库仅本地访问，无网络暴露
- 使用参数化查询（sqlx）防止 SQL 注入
- 前端输入校验（金额非负、必填项检查）
- 登录态使用本地安全存储（操作系统的 credential store 或加密本地文件）

### Compatibility
- macOS 12+ (ARM64 / x64)
- Windows 10/11 (x64)

### Usability
- 界面语言：简体中文
- 所有操作提供即时反馈（成功/失败提示）
- 金额、日期等字段使用合适的输入控件（数字键盘、日期选择器）

## 8. Acceptance Criteria

### AC1: 初始化向导
**Given** 应用首次启动（数据库中无用户），**When** 应用加载完成，**Then** 自动跳转到初始化向导页面，用户必须创建管理员账号（用户名 + 密码 + 确认密码）才能继续。

### AC2: 用户登录
**Given** 数据库已有用户账号，**When** 用户输入正确的用户名和密码并点击登录，**Then** 登录成功进入首页看板。**Given** 用户名或密码错误，**When** 点击登录，**Then** 提示"用户名或密码错误"并停留在登录页。

### AC3: 登录态持久化
**Given** 用户登录时勾选了"记住我"，**When** 关闭应用后重新打开，**Then** 在有效期内自动跳过登录页直接进入首页。

### AC4: 账本管理
**Given** 用户已登录，**When** 创建一个新账本（输入账本名称），**Then** 新账本出现在账本列表中，用户可以切换到该账本开始记账。

### AC5: 收入记录录入
**Given** 用户在某账本中，**When** 用户点击"新增记录"并填写必填项（日期、类别、总金额）后保存，**Then** 记录出现在该账本列表中，自动显示为"未结清"状态。

### AC6: 图片上传
**Given** 用户正在新增/编辑一条记录，**When** 用户上传多张图片，**Then** 图片保存到本地存储目录，记录详情中可查看缩略图，点击可查看原图。

### AC7: 结算操作
**Given** 某记录状态为"未结清"，**When** 用户点击"标记结清"→ 选择收款日期和收款方式 → 确认，**Then** 记录状态变为"已结清"，且编辑/删除按钮置灰。

### AC8: 已结清锁定
**Given** 某记录状态为"已结清"，**When** 用户尝试编辑该记录，**Then** 系统阻止操作并提示"已结清记录不可修改"。如需修改必须先回退为"未结清"。

### AC9: Excel 账单导出
**Given** 某账本中有未结清记录，**When** 用户全选或勾选部分未结清记录 → 点击"导出账单" → 选择保存路径，**Then** 生成一份 Excel 文件，包含所选记录的明细（日期、类别、描述、金额、备注）。

### AC10: 首页看板
**Given** 用户已登录在首页，**When** 查看看板，**Then** 显示：本月收入总额、当前未结清总额、各账本未结清金额排名（图表）。

### AC11: 数据备份与恢复
**Given** 用户点击"数据备份"，**When** 选择目标文件夹并确认，**Then** 数据库文件被复制到该文件夹。**Given** 用户点击"数据恢复"，**When** 选择备份文件并确认覆盖，**Then** 当前数据被替换为备份数据。

- ❌ 规格模板库（纸张类型、单价模板等）
- ❌ 部分付款 / 分期付款
- ❌ 客户详细资料（联系人、电话、公司地址等）
- ❌ 发票开具
- ❌ 利润分析（收入 - 支出）
- ❌ 打印/导出 PDF 账单
- ❌ 移动端（手机 App）
- ❌ 审计日志 / 操作历史
- ❌ 收款提醒 / 催款通知
- ❌ 汇率 / 多币种
